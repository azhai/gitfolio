1 快速开始 — 注册与登录
登录页面示意

🔐 注册新账号

  1. 打开 GitFolio 首页，点击「注册」
  2. 填写用户名、邮箱和密码
  3. 点击「创建账号」完成注册

🔑 登录系统

  1. 在登录页输入用户名和密码
  2. 点击「登录」进入仪表盘
  3. 登录后可在右上角头像菜单中修改个人资料
首次登录后建议先完善个人资料，上传头像方便团队识别。

2 创建与管理仓库
创建仓库 vs 迁移项目

📦 创建新仓库

  1. 点击左侧导航栏「项目」或仪表盘的「新建项目」
  2. 填写仓库名称（如 my-app）
  3. 选择可见性：公开 / 私有
  4. 可选：添加描述、初始化 README
  5. 点击「创建项目」

🔄 从 GitHub 导入

  1. 点击「迁移项目」
  2. 输入 GitHub 仓库 URL
  3. 系统自动拉取代码和元数据

⚙️ 仓库设置

  • 修改名称、描述和可见性
  • 管理仓库成员权限
  • 配置 Webhook
  • 删除仓库（需二次确认）

📂 文件浏览器

  • 进入仓库后默认显示文件列表
  • 点击文件夹进入子目录，点击文件查看内容
  • 文件列表显示最近提交信息和时间
  • README.md 自动渲染在文件列表下方

📝 查看提交历史

  • 点击「提交」标签查看完整提交记录
  • 点击某个提交查看详细差异
  • 支持按分支筛选提交
  • 提交详情页显示变更文件和 diff

🌿 分支与标签

  • 「分支」标签页列出所有分支
  • 「标签」标签页列出所有版本标签
  • 切换分支查看不同版本的代码
在文件浏览器中，点击提交信息的 hash 值可以快速跳转到该提交的详情页。

📋 创建 Issue

  1. 进入仓库，点击「Issues」标签
  2. 点击「新建 Issue」按钮
  3. 填写标题和描述（支持 Markdown）
  4. 可选：添加标签、指派人、关联里程碑
  5. 点击「提交」

🏷️ 标签管理

  • 系统预置常用标签：bug、feature、enhancement 等
  • 可自定义标签名称和颜色
  • 在 Issue 列表页可按标签筛选

💬 评论与互动

  • 在 Issue 详情页底部添加评论
  • 评论支持 Markdown 格式
  • 可引用其他 Issue：#123

5 Pull Request 协作
🟢 Open → 🔍 Review → ✅ Merged / ❌ Closed
PR 合并流程：feature 分支 → main 分支

🔀 创建 PR

  1. 点击「Pull Requests」→「新建 PR」
  2. 选择源分支和目标分支
  3. 填写标题和描述
  4. 可选：关联 Issue、指派审查人、添加标签
  5. 点击「创建 Pull Request」

👀 审查与合并

  • 查看 PR 的文件变更和 diff
  • 在代码行添加评论
  • 仓库所有者可点击「合并」完成合并
  • 合并后关联的任务自动流转到 Review 状态
PR 合并时，系统会自动检查关联的任务并将其状态流转为 Review，无需手动更新。

📋 创建任务

  1. 进入仓库，点击「Tasks」标签
  2. 点击「新建任务」
  3. 填写标题、描述、优先级（1-5）
  4. 可选：设置排期、关联 Issue
  5. 点击「创建」

⏱️ 时间追踪

  1. 在任务详情页点击「开始计时」
  2. 工作完成后点击「停止计时」
  3. 查看时间记录和汇总统计

🔗 关联管理

  • 关联 PR：将任务与代码变更关联
  • 关联 Issue：将任务与需求/缺陷关联
  • 查看关联提交：了解代码进展
  • 添加评论和附件

🔄 状态流转

  • 手动流转：点击「状态流转」选择目标状态
  • 自动流转：PR 合并时关联任务自动变为 Review
  • 查看流转历史：了解状态变更记录

7 提交关联与自动流转
提交消息中的关联语法与自动流转效果

🔗 提交消息中的关联语法

在提交消息中使用特殊关键词，系统会自动创建关联：

  • Fixes #123 — 关联并自动关闭 Issue #123
  • Closes #456 — 同上，关闭 Issue
  • Resolves #789 — 关联 Issue
  • References #42 — 仅引用，不关闭
  • Task: #5 — 关联任务 #5
  • PR #10 — 关联 PR #10

⚡ 自动流转规则

  • 提交消息含 Closes #123 → Issue #123 自动关闭
  • PR 合并 → 关联任务自动流转到 Review 状态
  • 所有关联记录可在提交详情页查看
示例提交消息：feat: add user avatar upload, Fixes #42, Task: #7 — 这将同时关联 Issue #42 和 Task #7。

8 团队与组织
团队结构示意

👥 创建团队

  1. 点击左侧导航「团队」
  2. 点击「新建团队」
  3. 填写团队名称和描述
  4. 上传团队头像

🤝 管理成员

  • 在团队详情页点击「添加成员」
  • 输入用户名添加新成员
  • 可移除不再需要的成员

📊 其他功能

  • ⭐ Star — 收藏感兴趣的仓库
  • 👁️ Watch — 关注仓库动态
  • 🏷️ 里程碑 — 管理版本计划
  • 📋 代码片段 — 保存常用代码
  • 🚀 Release — 管理版本发布
